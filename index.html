    Introduction

    Welcome to the Web of Registries! You can use this API to network together genetic design and data repositories. It currently only supports instance of SynBioHub, but support for other registries like ICE. The current authoritative Web of Registries is accessible at wor.synbiohub.org.

    The Web of Registries is, at its core, a curated list of data repositories. Anyone can submit their registry to the Web, but all registries must be approved by a Web of Registries administrator. To see a list of current administrators, please see our README.

    You can view example code in JavaScript or Python. The Python examples depend on a package called requests. The JavaScript examples depend on a package called request.

    If you would like more code example languages, or you find an error in the docs, please log an issue.

    Registries

    Get All Registries

    import requests

requests.get('https://wor.synbiohub.org/instances/');

    const request = include('request');

request.get('https://wor.synbiohub.org/instances/');

    The above command returns JSON structured like this:

    [
  {
    "id": 1,
    "uriPrefix": "http://synbiohub.org",
    "instanceUrl": "http://synbiohub.org",
    "description": "The main SynBioHub Instance",
    "name": "SynBioHub"
  },
  {
    "id": 2,
    "uriPrefix": "http://synbiohub.org",
    "instanceUrl": "http://dev.synbiohub.org",
    "description": "The development instance of SynBioHub",
    "name": "SynBioHub-dev"
  }
]

    This endpoint retrieves all approved data registries.

    HTTP Request

    GET /instances/

    Result Fields

    Parameter Type Description
    id integer The ID of the registry, unique for each registry.
    uriPrefix string (URI) The URI prefix of the registry -- no two registries will have the same uriPrefix.
    instanceUrl string (URL) The URL of the registry.
    description string The description written by the administrators of the registry. This is the same as the frontmatter of a SynBioHub instance.
    name string The name set by the administrators of the registry.

    This endpoint retrieves a specific registry.

    HTTP Request

    GET /instances/<instance_id>/

    Query Parameters

    Parameter Required Description
    instance_id yes The id of the instance to retrieve

    Result Fields

    Parameter Type Description
    id integer The ID of the registry, unique for each registry.
    uriPrefix string (URI) The URI prefix of the registry -- no two registries will have the same uriPrefix.
    instanceUrl string (URL) The URL of the registry.
    description string The description written by the administrators of the registry. This is the same as the frontmatter of a SynBioHub instance.
    name string The name set by the administrators of the registry.

    Register a New Registry

    import requests

registryData = {
  uriPrefix: "https://synbiohub.org/",
  instanceUrl: "https://synbiohub.org/",
  administratorEmail: "admin@synbiohub.org",
  updateEndpoint: "updateWebOfRegistries",
  name: "SynBioHub",
  description: "The reference SynBioHub instance",
}

requests.post('https://wor.synbiohub.org/instances/new/', data=registryData);

    const request = include('request');

let registryData = {
  uriPrefix: "https://synbiohub.org/",
  instanceUrl: "https://synbiohub.org/",
  administratorEmail: "admin@synbiohub.org",
  updateEndpoint: "updateWebOfRegistries",
  name: "SynBioHub",
  description: "The reference SynBioHub instance",
}

request.get('https://wor.synbiohub.org/instances/new/', { json: registryData });

    The above command returns JSON structured like this:

    {
  "id": 1,
  "uriPrefix": "https://synbiohub.org/",
  "instanceUrl": "https://synbiohub.org/",
  "administratorEmail": "admin@synbiohub.org",
  "updateEndpoint": "updateWebOfRegistries",
  "name": "SynBioHub",
  "description": "The reference SynBioHub instance",
  "updateSecret": "28501608beb2f0ab001d6f94fe9301d7469cf87bbc71b874e734da25f796b0c9d9ccbda5cd5f7f58b9331d7dfb08f8b"
}

    To register a SynBioHub instance, you should POST the necessary information to the Web of Registries. The updateEndpoint will immediately be sent the current state of the Web of Registries.

    Your SynBioHub instance must be approved by a Web of Registries administrator before it will be federated with the rest of the Web. The administrator may email you for more information about your SynBioHub instance. You can also check the status of the registration in the admin dashboard of SynBioHub. There, you will also be able to check the status of any updates sent by the Web of Registries -- if there is a communication issue, you will see an alert.

    HTTP Request

    POST /instances/new/

    Query Parameters

    Parameter Required Description
    uriPrefix yes
    instanceUrl yes
    administratorEmail yes
    updateEndpoint yes
    name yes
    description yes

    Result Fields

    Parameter Type Description
    id integer The ID of the registry, unique for each registry.
    uriPrefix string (URI)
    instanceUrl string (URL)
    administratorEmail string (email)
    updateEndpoint string
    name string
    description string
